## Convention ✅
### Branch Naming Convention 🪵
| 머릿말  | 설명                   |
| ------- | ---------------------|
| master  | 서비스 브랜치              |
| develop | 배포 전 작업 기준           |
| feature | 기능 단위 구현             |
| hotfix  | 서비스 중 긴급 수정 건에 대한 처리 |


### Commit Convention 🚥
| 머릿말   | 설명                      |
| -------- |-------------------------|
| feat     | 기능 구현, 추가               |
| setting  | 패키지 설치, 개발 설정           |
| refactor | 코드 리팩터링                 |
| style | 스타일 수정                  |
| fix      | 버그 수정, 예외 케이스 대응, 기능 개선 |
| test     | 테스트 코드 추가               |
| docs    | README.md작성 및 변경        |
